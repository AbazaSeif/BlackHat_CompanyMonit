Read First

The Basic_MT_C_S.zip file contains the following files:

  1. Compile_C_S.bat - a batch file that when double clicked will compile the MultiThreaded Client and Server programs.
  2. MT_C_S_Help.html - this file provides help on how to get started.
  3. tcpClient.java - the Java source file for the tcpClient.java program.
  4. tcpServer.java - the Java source file for the tcpServer.java program.
  5. MT_C_S_Tutorial.html - a file that describes how to build the MultiThreaded Client Server program.
  6. run_Javadoc.bat - the batch file that executes the Javadoc file for the tcpClient.java and tcpServer.java files.
  7. Start_Server.bat - when double-clicked will start the execution of the tcpServer.java program.
    • When running this application, this program should always be started first.
  8. Start_Client.bat - when double-clicked will start the execution of the tcpClient.java program.

How to compile and run

There are two ways to run the programs found in this .zip file.

  1. Multithreaded client Server program:
    1. You can double-click on the Compile_C_S.bat file that is supplied with the Basic_MT_C_S.zip file.
    2. You can load the tcpServer.java and tcpClient.java files into an IDE and run the programs from that environment.

A Basic Test Case

In this section, I have included a test case for the

  1. tcpClient.java and tcpServer.java programs.
    1. Double clicked on the Compile_C_S.bat file and this will compile the MultiThreaded tcpClient and tcpServer programs.
    2. To start executing the multithreaded client server application, do the following:
      • Start_Server.bat - when double-clicked will start the execution of the tcpServer.java program.
        • Always start the server first.
      • Start_Client.bat - when double-clicked will start the execution of the tcpClient.java program.
      • Start_Client.bat - when double-clicked will start the execution of the tcpClient.java program.
        • These actions will cause the server to start executing and there will be two clients executing.
    3. Enter This is a test... in the first client's test area and press the Send push button.
      • The server will send the message back along with the time it received the message. It will look something like: 
          
Time 9:40:0 From Server on Thread1 and message is This is a test... from Client4282
    4. Enter This is also a test... in the second client's test area and press the Send push button.
      • The server will send the message back along with the time it received the message. It will look something like: 
          
Time 9:40:0 From Server on Thread2 and message is This is also a test... from Client4283
    5. Press the Clear Message push button on the first client. This will clear the text area.
    6. Press the Clear Message push button on the second client. This will clear the text area.
  2. To enable the debugging messages, change the following line 62 from 
      
private boolean debug = false
    to 
      
private boolean debug = true
